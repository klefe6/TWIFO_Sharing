DAILY VIEW SPLIT-SCREEN ARTICLE VIEW - SUMMARY
================================================================================

FEATURE
-------
Added split-screen view for individual articles in Daily View.
Users can now see AI summary and original article side-by-side.

LAYOUT
------
Left Pane (45%):  AI-generated summary
Right Pane (55%): Original article PDF or fallback message

SCOPE
-----
MODIFIED:  Daily View article detail rendering only
UNCHANGED: Daily Summary rollup page (completely untouched)

================================================================================
IMPLEMENTATION
================================================================================

NEW FUNCTIONS (3)
-----------------

1. _find_original_pdf(artifact_folder: str) -> Optional[str]
   - Locates original PDF in artifact folder
   - Tries: original.pdf, source.pdf, article.pdf, any non-sum.pdf
   - Returns relative path for URL generation
   - Security: Only searches within artifact folder

2. _render_original_article_pane(artifact_folder: str, original_pdf_path: Optional[str]) -> html.Div
   - Renders right pane content
   - If PDF found: Shows iframe with embedded PDF + "Open in New Tab" link
   - If PDF not found: Shows fallback message in yellow warning box

3. _build_split_screen_layout(summary_content: html.Div, artifact_folder: str) -> html.Div
   - Wraps summary in split-screen layout
   - Creates flex container with two panes
   - Responsive: Stacks vertically on narrow screens (< 800px)

MODIFIED CALLBACK (1)
---------------------

display_daily_article_summary()
   - Added split-screen wrapper to 3 return statements:
     1. Strategy 1: sum.json rendering (line ~6369)
     2. Strategy 2: sum.pdf rendering (line ~6408)
     3. Strategy 3: No summary available (line ~6461)
   
   - Changed from:
       return art["artifact_folder"], content
   
   - Changed to:
       split_view = _build_split_screen_layout(content, art["artifact_folder"])
       return art["artifact_folder"], split_view

LAYOUT SPECS
------------
Left Pane:
  - Width: 45%
  - Min-width: 400px
  - Height: calc(100vh - 200px)
  - Overflow-y: auto
  - Border-right: 1px solid #dee2e6

Right Pane:
  - Width: 55%
  - Min-width: 400px
  - Height: calc(100vh - 200px)
  - Overflow-y: auto
  - Padding-left: 10px

Container:
  - Display: flex
  - Gap: 10px
  - Flex-wrap: wrap (stacks on narrow screens)

RESPONSIVE BEHAVIOR
-------------------
Desktop (> 800px):  Side-by-side panes
Mobile (< 800px):   Stacked vertically (summary on top, original below)

PDF DETECTION STRATEGY
----------------------
1. Try common names: original.pdf, source.pdf, article.pdf
2. Fallback: Any PDF that's not sum.pdf
3. If none found: Show fallback message
