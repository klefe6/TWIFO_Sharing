================================================================================

PROBLEM
-------
When a date had zero articles:
  - Left panel showed "No articles found"
  - Right panel showed STALE CONTENT from previous date
  - User experience was confusing and inconsistent

ROOT CAUSE
----------
The right panel callback only triggered on button clicks, not on date changes.
When the date changed to one with no articles, the left panel updated but the
right panel remained frozen.

SOLUTION
--------
1. Added daily-articles-store as an Input (not just State) to the right panel callback
2. Added early return logic when triggered by store update with empty artifacts
3. Added daily-view-date-input as State to show correct date in empty message

IMPLEMENTATION
--------------
File: twifo.py

Changed callback signature:
  - Added Input("daily-articles-store", "data")
  - Added State("daily-view-date-input", "value")
  - Removed State("econ-dynamics-mode", "data") [from previous task]

Added trigger differentiation logic:
  - If triggered by store + empty artifacts -> Clear right panel with empty state
  - If triggered by store + non-empty artifacts -> PreventUpdate (no flicker)
  - If triggered by button click -> Normal rendering (unchanged)

STORES CLEARED
--------------
1. daily-selected-artifact -> Set to "" (no selection)
2. daily-view-content -> Set to empty state HTML

Stores NOT cleared (and why):
- daily-articles-store: This is the INPUT, set by populate_daily_view_sidebar
- login-user: Authentication state, unrelated to articles
- econ-dynamics-mode: Removed in previous task (Daily View has no toggle)
