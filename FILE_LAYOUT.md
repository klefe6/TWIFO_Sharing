# TWIFO File Layout Documentation

**Last Updated:** 2026-02-12

## Overview

TWIFO now uses a **strict file layout** to separate original PDFs from generated artifacts. This prevents accidental overwrites and provides better organization.

---

## Directory Structure

```
FILES_DIR/
├── originals/              # Original PDFs (read-only, never overwrite)
│   ├── BOA_Report_20260212_w.pdf
│   ├── JPM_Analysis_20260211_m.pdf
│   └── ...
│
├── artifacts/              # Generated files (organized by basename)
│   ├── BOA_Report_20260212_w/
│   │   ├── extracted.txt   # Raw extracted text
│   │   ├── extraction.json # Extraction metadata
│   │   ├── sum.json        # Summary JSON (twifo.sum.v1)
│   │   ├── sum.txt         # Summary text
│   │   └── sum.pdf         # Rendered summary PDF
│   │
│   ├── JPM_Analysis_20260211_m/
│   │   ├── extracted.txt
│   │   ├── extraction.json
│   │   ├── sum.json
│   │   ├── sum.txt
│   │   └── sum.pdf
│   └── ...
│
└── rollups/                # Daily and weekly rollups
    ├── daily/
    │   ├── ROLLUP_DAILY_20260212__sum.json
    │   ├── ROLLUP_DAILY_20260212__sum.txt
    │   └── ROLLUP_DAILY_20260212__sum.pdf
    └── weekly/
        ├── ROLLUP_WEEKLY_20260210__sum.json
        ├── ROLLUP_WEEKLY_20260210__sum.txt
        └── ROLLUP_WEEKLY_20260210__sum.pdf
```

## File Descriptions

### Originals Directory (`originals/`)

- **Purpose:** Store original PDFs downloaded from Dropbox
- **Naming:** `{PREFIX}_{Title}_{YYYYMMDD}_{freq}.pdf`
  - Example: `BOA_Economic_Weekly_20260212_w.pdf`
- **Protection:** Read-only; never modified or overwritten
- **Managed by:** `import_dropbox.py`, `db_filter_autorun.py`

### Artifacts Directory (`artifacts/<basename>/`)

Each original PDF gets its own artifact directory containing:

| File | Description | Generated By |
|------|-------------|--------------|
| `extracted.txt` | Raw text extracted from PDF | `summarize_pdf.py` |
| `extraction.json` | Extraction metadata (method, quality, etc.) | `summarize_pdf.py` |
| `sum.json` | Structured summary (twifo.sum.v1 schema) | `summarize_pdf.py` (LLM) |
| `sum.txt` | Human-readable summary text | `summarize_pdf.py` |
| `sum.pdf` | Professional PDF summary | `summary_render.py` |

### Rollups Directory (`rollups/`)

- **Daily rollups:** `rollups/daily/ROLLUP_DAILY_YYYYMMDD__sum.*`
- **Weekly rollups:** `rollups/weekly/ROLLUP_WEEKLY_YYYYMMDD__sum.*`
- Managed by `generate_rollup_clean.py`, `run_weekly_rollup.py`

## Metadata in sum.json

All summary JSON files now include:

```json
{
  "schema_version": "twifo.sum.v1",
  "meta": {
    "original_pdf_path": "/path/to/originals/BOA_Report_20260212_w.pdf",
    "original_pdf_sha256": "abc123...",
    "prompt_version": "1.1",
    "prompt_sha256": "def456...",
    "extraction": {
      "status": "ok",
      "method_used": "pypdf",
      "extraction_quality_0_100": 95
    }
  },
  "sections": { ... }
}
```

## Components Updated

### Core Modules

| Module | Changes | Backward Compatible? |
|--------|---------|---------------------|
| `path_manager.py` | **NEW** - Centralized path management | N/A |
| `summarize_pdf.py` | Uses path_manager; adds metadata | ✅ Yes (via `path_manager` param) |
| `import_dropbox.py` | Downloads to `originals/` | ✅ Yes |
| `db_filter_autorun.py` | Uses path_manager; checks both layouts | ✅ Yes |
| `twifo.py` | Scans `originals/` and `artifacts/` | ✅ Yes |
| `summary_render.py` | No changes needed | ✅ Yes |

### Backward Compatibility

The system supports **BOTH** legacy and new layouts:

- **Legacy:** PDFs and `__sum.*` files in root `FILES_DIR`
- **New:** PDFs in `originals/`, artifacts in `artifacts/<basename>/`

All modules check the new layout first, then fall back to legacy paths.

## Migration

### Automatic Migration

Use the migration script to move existing files:

```bash
# Dry run (preview changes)
python migrate_file_layout.py --dry-run

# Live migration (creates backup first)
python migrate_file_layout.py

# Skip backup (not recommended)
python migrate_file_layout.py --no-backup

# Verify migration
python migrate_file_layout.py --verify-only
```

### Manual Migration

If needed, manually organize files:

1. Move PDFs (not `__sum.*`) to `originals/`
2. For each PDF basename, create `artifacts/<basename>/`
3. Move `<basename>__sum.json` → `artifacts/<basename>/sum.json`
4. Move `<basename>__sum.txt` → `artifacts/<basename>/sum.txt`
5. Move `<basename>__sum.pdf` → `artifacts/<basename>/sum.pdf`
6. Move `<basename>__sum_debug_raw.txt` → `artifacts/<basename>/extracted.txt`

## Benefits

### 1. **Safety**
- Original PDFs are never overwritten
- Clear separation between sources and artifacts

### 2. **Organization**
- All artifacts for a PDF in one directory
- Easy to find extracted text, JSON, PDF summary

### 3. **Traceability**
- `original_pdf_sha256` ensures integrity
- `original_pdf_path` links back to source

### 4. **Debugging**
- `extracted.txt` preserves raw extraction
- `extraction.json` tracks extraction metadata
- `sum_debug_raw.txt` no longer needed (use `extracted.txt`)

### 5. **Scalability**
- Flat directory structure avoided (no 10,000+ files in one folder)
- Easier to manage and backup

## Troubleshooting

### Issue: "Original PDF not found"

**Cause:** PDF hasn't been migrated to `originals/`

**Fix:**
```bash
python migrate_file_layout.py
```

### Issue: "Summary not showing in UI"

**Cause:** Path manager not finding artifacts

**Fix:** Check that artifacts exist:
```python
from path_manager import get_path_manager
pm = get_path_manager()
basename = "BOA_Report_20260212_w"
print(pm.has_summary(basename))  # Should return (True, True, True) if all exist
```

### Issue: "Old __sum.* files still in root"

**Cause:** Migration incomplete

**Fix:** Run migration again (safe to re-run):
```bash
python migrate_file_layout.py
```
